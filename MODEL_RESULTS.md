## 1. 프로젝트 개요 및 목표 설정

### 풀려는 문제

반도체 산업은 극심한 경기 사이클을 가집니다. 호황(Bull)과 불황(Bear)이 빠르게 교차하며, 기업들은 설비 투자·재고 전략·인력 계획을 6개월~1년 전에 결정해야 합니다. 이 프로젝트는 **현재 시점(T)의 데이터만으로 6개월 후(T+6) 글로벌 반도체 매출 YoY%를 예측**하는 것을 목표로 합니다.

### 타겟 변수

```
TARGET_Worldwide_YoY_T6   = Worldwide 반도체 매출 YoY% (6개월 후)
TARGET_Asia_Pacific_YoY_T6 = Asia Pacific 매출 YoY% (6개월 후)
```

YoY%(전년 동월 대비 성장률)를 타겟으로 선택한 이유:
- 절대 매출액은 물가·환율의 영향을 받아 시계열이 비정상(non-stationary)
- YoY%는 계절성과 장기 트렌드를 상당 부분 제거해 모델이 사이클 신호에 집중할 수 있음
- **방향(양수/음수) 분류 정확도**가 투자·경영 의사결정의 핵심 지표

### 예측의 비대칭성

반도체 산업에서 Bull 예측 실패보다 **Bear 예측 실패(불황 미감지)의 비용이 훨씬 큽니다.** 재고 과잉, 설비 과투자, 인력 구조조정으로 이어지기 때문입니다. 이를 반영하여 Bear 국면(YoY% < 0) 오차에 **1.5× 가중치**를 부여한 비대칭 손실 함수를 평가에 함께 사용합니다.

## 3. 데이터 범위 결정 과정

### 왜 1986년이 아닌 1993년부터 사용하는가

파이프라인의 학습 데이터는 **1993년 1월부터** 시작합니다. 두 가지 이유입니다.

**이유 1 — Asia Pacific 데이터 안정화**  
1986~1992년 구간의 Asia Pacific 지역 매출은 집계 방식이 현재와 달라 품질이 불안정합니다. Asia Pacific은 현재 반도체 시장의 60% 이상을 차지하는 핵심 지역이므로, 이 데이터의 품질 문제가 모델 전체에 영향을 줍니다.

**이유 2 — 병합 후 유효 데이터 시점**  
FRED 거시지표와 yfinance 주가 데이터는 1990년대 초 이전 구간에서 결측이 많습니다. 병합 후 모든 소스가 완전하게 확보되는 시점이 1993년입니다.

**결론**: 1986~1992년 데이터를 포함하면 7년치(84개월)를 추가할 수 있지만, Asia Pacific 품질 문제와 거시지표 결측으로 인해 현재는 1993년을 기준으로 유지합니다.

```
WSTS 파일 수록 기간:  1986-01 ~
FRED/주가 안정 시점:  ~1993-01
실제 학습 데이터:     1993-01 ~ 2025-07  (385개월)
```

## 4. 피쳐 엔지니어링

원본 데이터(397개월 × 24개 컬럼)에서 **136개 피쳐**를 생성합니다.

### 4-1. YoY% 변환

모든 레벨 데이터(WSTS 매출, FRED 거시지표)는 먼저 YoY%(전년 동월 대비 성장률)로 변환합니다.

```
YoY%(t) = (값(t) - 값(t-12)) / |값(t-12)| × 100
```

이유: 레벨 값은 장기 우상향 트렌드를 가져 비정상 시계열입니다. YoY% 변환으로 정상성을 확보하고 계절성도 제거합니다.

> 예외: `FRED_T10Y2Y`(장단기 금리차)는 이미 차이값(spread)이므로 레벨 그대로 사용합니다.

### 4-2. Lag 피쳐 — lag6, lag12만 사용

```
lag6(t)  = YoY%(t-6)   ← 6개월 전 값
lag12(t) = YoY%(t-12)  ← 1년 전 값 (동일 계절 비교)
```

**lag1~lag5를 제외한 이유:**  
예측 지평이 T+6이므로, lag1~lag5는 타겟(T+6)과 시간적으로 너무 가깝습니다. 실제로 계산해보면 lag1~lag3는 이동평균(ma3)과 상관계수가 **0.99** 수준으로, 중복 정보만 늘려 다중공선성을 악화시킵니다. lag6은 "정확히 예측 지평만큼 전의 상태", lag12는 "1년 전 동일 계절의 상태"로 각각 의미 있는 선행 정보를 담습니다.

### 4-3. 이동평균 — ma3, ma6, ma12

```
ma3(t)  = 최근 3개월 평균  ← 단기 추세
ma6(t)  = 최근 6개월 평균  ← 중기 추세
ma12(t) = 최근 12개월 평균 ← 장기 추세 방향
```

### 4-4. 변동성 — vol3, vol6

```
vol3(t) = 최근 3개월 표준편차  ← 단기 불확실성
vol6(t) = 최근 6개월 표준편차  ← 중기 불확실성
```

반도체 사이클에서 **전환점 직전에 변동성이 급등**하는 패턴이 있어, 이를 포착하기 위해 포함합니다.

### 4-5. 모멘텀, 가속도, 사이클 위치

| 피쳐 | 계산식 | 의미 |
|------|--------|------|
| `momentum_3_12` | ma3 - ma12 | 단기 추세가 장기 대비 강한지(+) 약한지(-) |
| `accel` | YoY%(t) - YoY%(t-1) | YoY%의 기울기 변화 — 전환점 감지 |
| `cycle_pct24` | 최근 24개월 내 Percentile Rank | 현재가 24개월 사이클 내 어느 위치인지 (0=바닥, 1=정점) |

### 4-6. 계절성 인코딩

월(1~12)을 사인·코사인으로 변환해 1월과 12월이 연속적으로 연결되도록 처리합니다.

```
month_sin = sin(2π × month / 12)
month_cos = cos(2π × month / 12)
```

### 4-7. 피쳐 구성 요약

| 카테고리 | 피쳐 수 | 대표 피쳐 |
|----------|---------|----------|
| WSTS 지역 YoY% + 파생 | 55개 | `Asia_Pacific_YoY`, `Worldwide_YoY_cycle_pct24` |
| 주가 수익률 + 파생 | 37개 | `Ret_SOX_ma6`, `Ret_ASML_vol6` |
| FRED 거시지표 YoY% + 파생 | 42개 | `FRED_IndProd_YoY_lag12`, `FRED_T10Y2Y_chg3` |
| 계절성 | 2개 | `month_sin`, `month_cos` |
| **합계** | **136개** | |

### 4-8. 데이터 누설 방지

모든 피쳐는 예측 시점 T까지의 과거 데이터만 사용합니다. 타겟은 `shift(-6)`으로 생성하므로 피쳐와 타겟 간 누설이 없습니다.

```
피쳐 (T 이전 정보만)  →  모델  →  예측: T+6 YoY%
```

## 5. 모델 선택 및 검증 전략

### 5-1. 모델 선택 이유

| 모델 | 선택 이유 |
|------|----------|
| **Ridge** | 다중공선성이 있는 피쳐셋에서 계수를 안정적으로 추정. 선형 베이스라인 역할 |
| **Lasso** | L1 정규화로 136개 피쳐 중 실제 유효한 피쳐를 자동 선택. 해석 가능성 확보 |
| **XGBoost** | 비선형 임계 효과·사이클 전환 패턴 포착. 반도체처럼 급격한 변화에 강함 |
| **LightGBM** | leaf-wise 분기로 세밀한 패턴 학습. XGBoost 대비 빠른 학습 속도 |

### 5-2. 시계열 교차검증 설계

일반적인 k-fold CV는 미래 데이터가 학습셋에 포함되는 **데이터 누설** 문제가 있습니다. 시계열에서는 반드시 **walk-forward validation**을 사용해야 합니다.

```
전체 시계열 (385개월)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Fold 1: [━━━━━━━━Train━━━━━━━━][━━Test(12m)━━]
Fold 2: [━━━━━━━━━━━Train━━━━━━━━━━][━━Test━━]
Fold 3: [━━━━━━━━━━━━━━Train━━━━━━━━━━━━][Test]
Fold 4: [━━━━━━━━━━━━━━━━━━Train━━━━━━━━━━][T.]
Fold 5: [━━━━━━━━━━━━━━━━━━━━━━━Train━━━━━━][T]
```

- **n_splits=5, test_size=12개월**: 각 fold의 테스트셋이 12개월(1년)
- **훈련셋은 항상 테스트셋보다 과거** 데이터만 포함
- 미래 누설 완전 차단

### 5-3. 평가 지표

| 지표 | 설명 | 선택 이유 |
|------|------|----------|
| **RMSE** | 예측 오차 제곱 평균의 제곱근 | 급락·급등 구간 오차에 민감 — 반도체 사이클에 적합 |
| **MAE** | 절대 오차 평균 | 이상치에 강건한 보조 지표 |
| **방향 정확도** | Bull/Bear 분류 정확도 | **투자·경영 의사결정의 핵심** |
| **비대칭 손실** | Bear 오차에 1.5× 가중 | Bear 미감지 페널티 강화 |
| MAPE | 절대 퍼센트 오차 | YoY%=0 근처에서 불안정 — 참고 지표로만 활용 |

## 7. 하이퍼파라미터 최적화

**Optuna(Bayesian 최적화, TPE Sampler)**를 사용해 모델당 50회 trial을 수행합니다.

### 7-1. 최적화 전후 성능 비교

| 모델 | 기본 RMSE | 최적화 후 RMSE | 개선폭 |
|------|----------|--------------|--------|
| **XGBoost** | 10.40 | **9.51** | -0.89 |
| **LightGBM** | 10.65 | **9.96** | -0.69 |
| Ridge | 27.46 | 12.74 | -14.72 |
| Lasso | 20.31 | 12.81 | -7.50 |

> Ridge/Lasso는 정규화 강도(alpha) 탐색으로 큰 폭 개선. 트리 모델은 기본값이 이미 준최적에 가까워 개선폭이 작음.

### 7-2. Trial 통계 (50 trials × 4 모델 = 200회)

## 9. 주요 인사이트

**인사이트 1 — 반도체 사이클은 12개월 패턴이 강하다**  
XGBoost에서 1위(23%)와 3위(14%)가 모두 `lag12` 피쳐입니다. 반도체 수요는 1년 전 산업 생산 수준 및 신규 수주와 강한 상관을 가지며, 이는 반도체 설계·제조·납품의 리드타임이 약 6~12개월임을 반영합니다.

**인사이트 2 — 주가(SOX)는 일관된 선행 지표다**  
`Ret_SOX_ma6`가 두 모델 모두 상위 11위 이내에 들어갑니다. 주식시장이 실물 경기를 선행하는 특성이 반도체 업황 예측에도 유효하게 작동합니다.

**인사이트 3 — 신규 추가 FRED 지표(MANEMP, UMCSENT, NEWORDER)가 실제로 기여한다**  
폐기된 NAPM(ISM PMI) 대체 목적으로 추가한 세 지표가 예상 이상으로 중요하게 나타났습니다:
- `FRED_NewOrder_YoY_lag12`: XGBoost **3위(14.4%)** — 가장 유효한 대체 지표
- `FRED_MfgEmp` 계열: XGBoost 5·9·10위 합산 **10.3%**
- `FRED_ConsSenti_YoY_lag6`: LightGBM **6위(6.7%)**

**인사이트 4 — 변동성 피쳐가 사이클 전환점을 감지한다**  
`vol3`, `vol6` 피쳐들이 두 모델 모두 상위권에 분포합니다. 반도체 사이클은 전환 직전에 지역별 매출 변동성이 급등하는 패턴이 있으며, 모델이 이를 학습한 것으로 해석됩니다.

**인사이트 5 — lag1~5 제거가 트리 모델 성능을 개선했다**  
단기 lag 제거 전후로 XGBoost RMSE가 10.44 → 9.51로 개선됐습니다. 타겟(T+6)과 너무 가까운 단기 lag가 오히려 과적합을 유도했음을 보여줍니다.

## 10. 한계 및 개선 방향

### 10-1. 선형 모델 성능 열위 — 다중공선성

**현상**: Ridge RMSE 12.74, Lasso 12.81 — XGBoost(9.51) 대비 30% 이상 열위

**원인 분석:**
- `ma3 ≈ (현재 + lag1 + lag2) / 3` — lag를 제거했어도 ma 안에 단기 정보가 잔류
- `Worldwide = Americas + Europe + Japan + Asia_Pacific` — 완전한 선형 종속
- 지역별 YoY% 시계열 간 높은 상관 (글로벌 경기 동조화)

**개선 방안**: `Worldwide_YoY` 관련 피쳐 전체 제거 후 VIF(분산팽창인수) > 10인 피쳐를 순차 제거하거나, 선형 모델 앞에 PCA를 전처리로 추가

### 10-2. MAPE 지표 불안정

**현상**: Lasso MAPE 146.8%, Ridge 162.1% — 수치가 비정상적으로 큼

**원인**: `MAPE = |실제 - 예측| / |실제| × 100` 에서 YoY%가 0에 가까울 때 분모가 0에 수렴해 폭발. 반도체 사이클에서 Bull→Bear 전환 구간이 정확히 이 문제 구간입니다.

**개선 방안**: sMAPE(분모를 `(|실제| + |예측|) / 2`로 대체) 또는 RMSSE(naive forecast 대비 스케일링) 사용

### 10-3. 데이터 볼륨 제한

**현상**: 학습 데이터 385개월 — 딥러닝 모델(N-HiTS, PatchTST 등) 학습에 부족

**원인 및 맥락:**
- WSTS 파일 자체가 1986년부터 수록 (1976~1985년은 별도 아카이브)
- 1986~1992년 데이터를 포함하면 84개월 추가 가능하나 Asia Pacific 품질 문제 존재

**개선 방안**: SEMI(반도체 장비 출하) 데이터 등 추가 소스로 피쳐 다양성 확대. 딥러닝은 현실적으로 현재 규모에서 트리 대비 열위이므로 트리 앙상블 집중 권장

### 10-4. 거시지표 발표 시차 — 실전 배포 시 가장 치명적

**현상**: 파이프라인이 각 지표의 해당 월 값을 즉시 사용 가능한 것으로 가정

**실제 문제**: 예를 들어 2025년 3월 예측 시, `INDPRO`(산업생산지수)는 4~6주 후 발표되므로 실제로는 1월치 데이터밖에 없습니다.

| 지표 | 실제 발표 시차 |
|------|--------------|
| WSTS 월별 매출 | 약 6~8주 후 |
| FRED_IndProd, FRED_MfgEmp | 약 4~6주 후 |
| FRED_NewOrder | 약 4주 후 |
| FRED_ConsSenti (UMCSENT) | 당월 말 (속보치 가능) |
| yfinance 주가 | 실시간 |

**개선 방안**: 각 시리즈에 발표 시차만큼 추가 shift 적용. 주가 데이터(실시간)를 주력으로, 거시지표는 보조 역할로 재설계
